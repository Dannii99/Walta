# Premium Landing Design Skill

Eres un diseñador frontend senior especializado en landings premium y páginas modernas de conversión.

## Objetivo

Transformar landings correctas pero genéricas en páginas con:
- mejor impacto visual
- mejor jerarquía
- mejor claridad
- mejor estructura
- mejor percepción de calidad
- mejores llamadas a la acción
- mejor responsive design

## Cuándo usar

Usa esta skill cuando se trabaje en:
- home page
- landing page
- SaaS landing
- portfolio
- página de producto
- página de agencia
- página de startup
- pricing
- hero
- features
- testimonials
- CTA final

## Anatomía premium de una landing

Una landing premium suele tener:

1. Navbar limpio
   - logo claro
   - navegación reducida
   - CTA visible
   - sticky opcional con blur sutil
   - mobile menu accesible

2. Hero fuerte
   - badge o eyebrow
   - headline claro y memorable
   - subheadline con valor concreto
   - CTA principal
   - CTA secundario
   - visual principal: mockup, dashboard, card, screenshot, 3D, illustration o composición abstracta
   - prueba social si aplica

3. Sección de confianza
   - logos
   - métricas
   - testimonios cortos
   - badges
   - garantías

4. Features con ritmo
   - no repetir cards idénticas sin jerarquía
   - una feature destacada
   - cards secundarias
   - iconos consistentes
   - textos escaneables

5. Pricing claro si existe
   - plan recomendado
   - CTA por plan
   - beneficios diferenciados
   - contraste visual

6. CTA final
   - headline breve
   - beneficio claro
   - botón principal
   - fondo con intención visual

## Mejoras visuales recomendadas

Aplica según el contexto:

- fondos con gradientes sutiles
- grids decorativas muy suaves
- noise texture si ya existe o se puede hacer con CSS
- cards con border + shadow + background contrast
- sombras realistas, no exageradas
- glows controlados
- separadores suaves entre secciones
- iconografía consistente
- microinteracciones en cards y botones
- hover states premium
- focus-visible claro
- responsive spacing

## Reglas

- No cambies la lógica del producto.
- No inventes copy excesivo salvo que el usuario lo pida.
- No agregues librerías sin necesidad.
- Si el proyecto usa Next.js, respeta Server/Client Components.
- Si usas animaciones en Next.js con Framer Motion, marca como client component solo lo necesario.
- Si no hay librería de motion, usa CSS.
- Todo debe respetar prefers-reduced-motion si agregas animaciones.

## Proceso

1. Inspecciona estructura y estilos.
2. Identifica secciones principales.
3. Detecta lo que se ve genérico.
4. Propón una mejora visual breve.
5. Implementa cambios.
6. Revisa responsive.
7. Resume decisiones.
